API Architecture
Guide

A strict, layered 6-tier architecture. Each layer builds upon the outputs of the previous, transforming raw unstructured documents into a mathematically grounded, conversational intelligence system.

Layer 1

Ingestion & Deduplication

Transforms diverse multi-modality sources into normalized text records, calculating SHA-256 hashes to prevent duplicate compute.

POST/projects

Creates a unique multi-document workspace container.

Request Body
{
  "name": "Quantum Computing Papers 2024",
  "description": "Optional metadata"
}
201 CreatedReturns a ProjectModel with a generated UUID (id).
POST/projects/{project_id}/ingest

Uploads binary files (PDF, DOCX, MD, TXT). Uses multipart/form-data.

Headers Override
Content-Type: multipart/form-data
Body
file  (binary)
201 CreatedReturns a DocumentSummary. processing_status will be "pending" — Layer 6 takes over background chunking/embedding.
POST/projects/{project_id}/ingest-repo

Recursively clones a public GitHub repo, extracting all READMEs and Python docstrings.

Request Body
{
  "repo_url": "https://github.com/GwadeSteve/VERO",
  "title": "Optional Override Title"
}
Layer 2

Reversible SOTA Chunking

Breaks massive text blocks down into token-optimized shards, preserving semantic boundaries and markdown hierarchy (breadcrumbs).

POST/documents/{doc_id}/chunkauto or manual

Triggered automatically by Layer 6, but available manually. Applies tiktoken limits alongside semantic sentence splitting or Markdown header awareness.

201 CreatedReturns an array of ChunkResponse.
GET/documents/{doc_id}/chunks
200 OK
[
  {
    "id": "e4f8b2",
    "text": "[Root > Installation]\nClone the repo...",
    "start_char": 450,
    "end_char": 890,
    "token_count": 112,
    "strategy": "markdown"
  }
]
Layer 3

Versioned Vector Embeddings

Generates 384-dimensional dense vectors using all-MiniLM-L6-v2 via sentence-transformers. Skips chunk hashes that haven't changed.

POST/documents/{doc_id}/embedauto or manual

Triggered automatically by Layer 6, but available manually. Calculates embeddings and upserts them into persistent ChromaDB storage.

201 Created
[
  {
    "chunk_id": "e4f8b2",
    "model_name": "all-MiniLM-L6-v2",
    "dimension": 384,
    "is_cached": false
  }
]
Layer 4

Hybrid Retrieval Pipeline

The heart of VERO's accuracy. Concurrent sparse and dense searches, fused via RRF, then sent through a Cross-Encoder neural network for precise relevance ranking.

Hybrid search · parallel BM25 + semantic → rerank

POST/projects/{project_id}/search
Request Body
{
  "query": "How is context preserved during chunking?",
  "top_k": 5,
  "mode": "hybrid"
}
mode: hybrid semantic keyword
POST/projects/{project_id}/search/context

A specialized endpoint that bundles the returned chunks into an optimal prompt-ready format for direct LLM injection.

Layer 5

Grounded Answering System

Replaces hallucination-prone generation with strict synthesis. The LLM acts purely as a summarizer over the retrieved search constraints.

POST/projects/{project_id}/answerstateless

One-shot / Stateless endpoint. Set "allow_model_knowledge": true to let the LLM supplement with training knowledge when documents are insufficient.

Request Body
{
  "query": "What formats does VERO support?",
  "top_k": 3,
  "allow_model_knowledge": false
}
200 OK
{
  "answer": "VERO supports PDF, DOCX, Markdown [Source 1]...",
  "citations": [
    {
      "chunk_id": "e4f8b2",
      "doc_id": "a1b2c3",
      "doc_title": "README.md",
      "start_char": 450,
      "end_char": 890,
      "score": 0.8412,
      "source_url": null
    }
  ],
  "found_sufficient_info": true
}

Each citation includes doc_id, chunk_id, start_char, and end_char for precise UI highlighting of the exact passage used.

Layer 6

Conversational Intelligence

Elevates VERO from a basic RAG system to an autonomous, context-aware memory agent with sliding-window history and full citation persistence.

Auto-Pipeline — Uploading a document in Layer 1 triggers a thread-pooled background task. Monitor GET /documents/{doc_id} and check the processing_status property.
pending→ processing→ ready
Memory flow · sliding window + retrieval + persist

POST/projects/{project_id}/sessions
Request Body
{ "title": "Architecture Research Chat" }
201 CreatedReturns SessionResponse.
GET/sessions/{session_id}

Returns the entire saved chronological conversation tree.

200 OK
{
  "id": "abc-123",
  "title": "Architecture Research Chat",
  "messages": [
    { "role": "user",      "content": "Tell me about Layer 2." },
    { "role": "assistant", "content": "Layer 2 handles chunking..." }
  ]
}
POST/sessions/{session_id}/chat

The pinnacle interaction point. Sends a message into an active session. The model automatically loads conversation history, resolving pronouns across turns — e.g. "How do I install it?" correctly resolves "it" from a prior message.

Request Body
{
  "message": "Can I replace the chunking strategy?",
  "top_k": 5,
  "mode": "hybrid",
  "allow_model_knowledge": false
}
200 OKReturns ChatResponse with answer, citations (incl. doc_id, chunk_id, start_char, end_char), and found_sufficient_info.
